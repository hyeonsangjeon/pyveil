# pyveil compatibility contract

This directory holds pyveil's machine-readable protection-surface contract and
the synthetic fixtures that back every claim in it. Nothing here contains real
personal data, live credentials, or personal filesystem paths.

- [`manifest.json`](manifest.json) â€” one entry per public `Channel`, with its
  status, install, entrypoint, capabilities, constraints, security boundary,
  the fixtures that verify it, and a reproducible verification command.
- [`fixtures.json`](fixtures.json) â€” synthetic prompt, tool-call, tool-result,
  MCP, memory, trace, and log payloads with expected redaction, expected
  preserved (non-sensitive) values, and fail-closed expectations.
- [`receipt.json`](receipt.json) / [`receipt.md`](receipt.md) â€” a privacy-safe
  Proof-of-Compatibility Receipt: per-surface entity counts, gate results, and
  output hashes, with no raw prompt, PII, or secret.

## Status semantics

A surface is only `verified` when it has policy support, at least one synthetic
fixture, and a passing verification command. Use `experimental` for a surface
under active work and `unsupported` for a channel pyveil does not cover. The
validator rejects any other status value.

## Reproduce

```bash
# 1. Validate the manifest, fixtures, and generated table above.
python scripts/validate_compatibility.py --check

# 2. Run the synthetic protection-surface tests.
python -m pytest tests/test_compatibility_surfaces.py

# 3. Regenerate the Proof-of-Compatibility Receipt.
python scripts/compatibility_receipt.py --write
```

`--write` regenerates the table (step 1) or the receipt (step 3) instead of
comparing. CI runs the `--check` forms so drift between the manifest, the
fixtures, and this table fails the build.

## Provenance

The contract, validator, and receipt are an independent Python implementation
of patterns observed while benchmarking
[`affaan-m/ECC`](https://github.com/affaan-m/ECC) (MIT), reviewed at commit
`fb401834e5b48679e1a6050e4bb86dce0824bbf5` on 2026-07-24. pyveil applies the
*ideas* â€” a machine-readable capability manifest, fixture-backed status,
generated-and-checked support tables, and a public evidence receipt â€” to its
own narrow redaction problem. No ECC code, workflow, branding, or wording was
copied, and no ECC popularity or SLSA claim is inherited.
